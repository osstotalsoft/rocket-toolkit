üéØ Universal Pattern: Event-Driven Recursion

All three operations follow the same recursive pattern:

  1. Wrap the event handler with additional logic
  2. When event arrives: Call inner handler(s) to process event
  3. Recursively apply the same operation to the resulting state
  4. Return transformed result with combined effects

This recursive wrapping creates a transformation pipeline where:

  • map transforms result values through the entire event chain
  • bind sequences processes by continuing the chain on success
  • apply processes events in parallel and merges states via tuple tracking

The magic is that one operation wraps the entire future computation, no matter how many events will arrive!
